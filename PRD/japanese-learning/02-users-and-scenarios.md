# 用户与场景

## 用户角色

### 日语学习者

目标：

- 输入日语句子并获得中文释义。
- 听日语原文发音。
- 将本次学习内容保存为资源。
- 查看今日和累计学习成果。

权限边界：

- 可以进入学习游戏。
- 可以提交学习结果。
- 可以查看学习统计。
- 是否可以维护全部资源，当前项目未确认。

### 学习资源维护者

目标：

- 新增、编辑、删除和查询日语学习资源。
- 为资源生成日语和中文朗读音频。
- 为资源补充中文释义。
- 导出资源或查看资源详情。

权限边界：

- 可以操作资源维护能力。
- 可以执行选中资源的批量处理。
- 是否需要审批、角色授权或操作审计，当前项目未确认。

### 内容整理者

目标：

- 按来源组织学习资源。
- 维护“日语基础”等基础资源集合。
- 通过卡片查看基础内容并检查音频。

权限边界：

- 可以访问基础卡片视图。
- 可以维护资源来源。
- 资源来源是否存在统一字典，当前项目未确认。

### 外部语言能力

目标：

- 根据日语文本返回中文释义。
- 为日语文本和中文文本提供朗读音频。

权限边界：

- 只作为语言处理能力参与业务流程。
- 其可用性、费用、隐私和失败策略需要在独立实现时明确。

## 关键场景

### 场景 S1：维护日语学习资源

前置条件：

- 用户有权访问日语学习资源维护入口。

用户目标：

- 维护日语文本、中文含义、来源和音频。

完成条件：

- 资源被成功创建、更新、删除或查询到。
- 操作后列表刷新并反映最新结果。

### 场景 S2：听单条日语资源

前置条件：

- 资源存在。
- 资源至少有一个可播放的日语朗读音频。

用户目标：

- 快速听到该条日语原文发音。

完成条件：

- 播放该资源的第一个日语朗读音频。

### 场景 S3：查看基础日语卡片

前置条件：

- 存在来源为“日语基础”的学习资源。

用户目标：

- 以卡片方式查看基础日语内容，并按卡片听音。

完成条件：

- 系统展示基础资源卡片矩阵。
- 用户可以点击卡片听音入口播放日语音频。

### 场景 S4：批量生成听力课程音频

前置条件：

- 用户选择一个或多个资源。
- 被纳入合成的资源同时具备日语朗读和中文朗读音频。

用户目标：

- 将所选资源合成为连续听力音频。

完成条件：

- 系统返回一个可播放或可下载的合成音频资源。

### 场景 S5：游戏式录入日语句子

前置条件：

- 用户进入学习游戏。
- 外部翻译和音频能力可用。

用户目标：

- 输入日语句子，获得翻译，听原文，并保存为学习资源。

完成条件：

- 本次句子被保存为学习资源。
- 今日和累计学习统计增加。
- 页面提示“学习成功”。

### 场景 S6：进入学习游戏查看统计

前置条件：

- 系统已有学习统计记录。

用户目标：

- 查看今日学习数量、今日学习时长、累计学习数量和累计学习时长。

完成条件：

- 如跨日，今日统计先重置。
- 页面展示最新统计值。
