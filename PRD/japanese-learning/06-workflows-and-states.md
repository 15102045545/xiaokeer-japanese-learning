# 流程与状态

## 流程一：日语资源维护

1. 用户进入日语资源维护视图。
2. 系统加载查询条件、资源列表、来源分组和可执行操作。
3. 用户执行以下操作之一：
   - 查询资源。
   - 新增资源。
   - 查看详情。
   - 编辑资源。
   - 删除资源。
   - 导出当前查询结果。
   - 执行批量音频或翻译处理。
4. 系统完成操作后刷新列表。

### 失败分支

- 查询失败：提示用户查询失败，保留当前页面。
- 保存失败：资源不应被部分保存，提示用户重试。
- 删除失败：未删除的资源仍应可见。
- 批量操作部分失败：成功项保留，失败项应可追踪；当前用户可见失败详情待确认。

### 取消分支

- 用户取消新增或编辑时，不保存表单内容。
- 用户取消批量操作确认时，不执行批量操作。

## 流程二：卡片查看与听音

1. 用户切换到基础卡片视图。
2. 系统加载基础来源的日语学习资源。
3. 系统以卡片矩阵展示资源，空位展示空卡片。
4. 用户点击卡片听音入口。
5. 系统播放该资源的第一个日语朗读音频。

### 状态

| 状态 | 说明 |
| --- | --- |
| 编辑模式 | 展示资源维护能力 |
| 卡片模式 | 展示基础资源卡片 |
| 可听音卡片 | 资源有日语音频 |
| 空卡片 | 矩阵位置无资源 |

### 失败分支

- 基础资源为空：展示空矩阵或空状态；具体表现待确认。
- 音频缺失：不播放或播放失败；当前缺少明确提示规则。

## 流程三：批量生成朗读音频

1. 用户选择一个或多个资源。
2. 用户确认执行朗读音频生成。
3. 系统逐条处理资源。
4. 系统基于日语文本生成日语朗读音频。
5. 系统基于中文含义生成中文朗读音频。
6. 系统把新音频追加到对应资源的音频列表。
7. 列表刷新。

### 状态

| 状态 | 进入条件 | 退出条件 |
| --- | --- | --- |
| 待生成 | 资源缺少音频或用户要求追加音频 | 开始处理 |
| 生成中 | 外部朗读能力正在处理 | 成功或失败 |
| 已追加 | 新音频写入资源音频列表 | 用户再次生成或编辑 |
| 生成失败 | 单条资源生成失败 | 用户重试 |

### 重试规则

- 用户可以再次对同一资源执行生成。
- 当前重试会追加音频，是否应避免重复待确认。

## 流程四：批量补充中文释义

1. 用户选择一个或多个资源。
2. 用户确认执行中文释义补充。
3. 系统逐条读取日语文本。
4. 系统获取中文翻译。
5. 系统将翻译结果写入资源的中文含义。
6. 列表刷新。

### 失败分支

- 日语文本为空：应跳过或提示；当前行为待确认。
- 翻译失败：应保留原中文含义；当前提示待确认。

## 流程五：生成听力课程音频

1. 用户选择一个或多个资源。
2. 用户执行课程音频生成。
3. 系统读取所选资源。
4. 系统过滤掉缺少日语音频或中文音频的资源。
5. 系统按资源顺序拼接双语训练片段。
6. 系统生成一个合成音频资源。
7. 系统返回合成音频资源信息。

### 双语训练片段状态

| 状态 | 说明 |
| --- | --- |
| 可纳入 | 资源同时具备日语音频和中文音频 |
| 被跳过 | 资源缺少任一必要音频 |
| 已拼接 | 资源音频已进入合成序列 |

### 失败分支

- 所选资源为空：应提示未选择资源。
- 所选资源均不可纳入：应提示缺少必要音频；当前提示待确认。
- 合成失败：应提示用户重试。

## 流程六：学习游戏录入

1. 用户进入学习游戏。
2. 系统刷新今日统计。
3. 系统展示今日统计和累计统计。
4. 用户在输入区录入日语句子。
5. 用户按回车。
6. 系统将输入文本翻译为中文，并获得日语原文音频和中文译文音频。
7. 系统展示处理中状态，显示原文和译文。
8. 系统播放日语原文音频一次。
9. 系统计量本次播放耗时。
10. 系统清空输入区并关闭处理中状态。
11. 系统提交学习结果。
12. 系统保存一条日语学习资源。
13. 系统更新累计统计和今日统计。
14. 系统刷新统计展示并提示学习成功。

### 学习提交状态机

| 状态 | 进入条件 | 可流转到 | 说明 |
| --- | --- | --- | --- |
| 待输入 | 用户进入游戏 | 翻译中 | 等待用户输入并回车 |
| 翻译中 | 用户提交输入 | 处理中、失败 | 获取翻译和音频 |
| 处理中 | 翻译和音频已返回 | 提交中、失败 | 展示原文译文并播放日语音频 |
| 提交中 | 音频播放结束 | 完成、失败 | 保存资源并更新统计 |
| 完成 | 保存和统计成功 | 待输入 | 清空输入并刷新统计 |
| 失败 | 任一步骤失败 | 待输入 | 用户可再次输入重试 |

### 失败分支

- 翻译失败：不应保存资源，也不应更新统计。
- 音频播放失败：是否仍允许保存待确认。
- 提交失败：不应更新统计，或应回滚资源保存；当前一致性策略待确认。
- 统计记录不存在：提示“未找到游戏统计信息”。

## 流程七：每日统计刷新

1. 用户进入学习游戏或提交学习结果。
2. 系统比较统计日期和当前日期。
3. 如果日期一致，保留今日统计。
4. 如果日期不一致，今日学习数量和今日学习时长归零，并将统计日期更新为当前日期。
5. 若当前触发点是学习提交，则在归零后累加本次结果。

### 状态

| 状态 | 说明 |
| --- | --- |
| 当日有效 | 统计日期等于当前日期 |
| 需刷新 | 统计日期早于或不等于当前日期 |
| 已刷新 | 今日统计归零且日期更新 |
| 已累加 | 成功提交后统计增加 |
